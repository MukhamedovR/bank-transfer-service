# Bank Transfer Service (Java 21 & Spring Boot)

Сервис асинхронной обработки банковских переводов между счетами с поддержкой мультивалютности. Разработан в соответствии с техническим заданием.

## 🛠 Технологический стек
* **Java 21** — использование **Virtual Threads (Виртуальных потоков)** как легковесной альтернативы горутинам в Go для параллельной обработки запросов.
* **Spring Boot 3.2.4** (Spring Web, Spring Data JPA).
* **База данных:** Встроенная СУБД H2 (поддерживается автоматическая генерация схем данных JPA).
* **Тестирование:** JUnit 5, Mockito (Unit-тестирование бизнес-логики и обработки исключений).

## 🚀 Архитектурные решения и особенности
1. **Асинхронность на Virtual Threads:** В настройках приложения активирован пул виртуальных потоков (`spring.threads.virtual.enabled=true`). Каждый перевод выполняется асинхронно через `CompletableFuture`, не блокируя основной поток сервера.
2. **Безопасность транзакций:** Логика списания и зачисления средств защищена аннотацией `@Transactional` для обеспечения атомарности (Rollback в случае ошибок).
3. **Мультивалютность (Дополнительное задание):** Реализована встроенная логика автоматической конвертации средств, если валюты счета отправителя и получателя отличаются (поддерживаются USD/EUR).
4. **Уведомления (Дополнительное задание):** Интегрирована асинхронная система логирования результатов операций.

## 🧪 Тестирование
В проекте реализованы автоматические Unit-тесты для проверки ключевых сценариев:
* Успешный перевод между счетами.
* Блокировка транзакции при недостатке средств (Insufficient funds).
* Корректность работы математического механизма конвертации валют.
